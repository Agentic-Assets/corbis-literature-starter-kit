Run the `paper-elevator` skill on this paper:

$ARGUMENTS

Read the paper (`.tex` file, or use the project state and user description). Classify the paper type, identification strategy, and target track. Inventory what tests already exist. Then produce a prioritized gap report covering:

1. **Must-have** tests (paper will be rejected without these)
2. **Expected** tests (referees will ask; preempting saves a revision round)
3. **Elevating** tests (signals top-journal quality and methodological awareness)
4. **Conceptual gaps** (missing model, mechanism structure, contribution precision, external validity)
5. **Not recommended** (tests that would NOT add value for this specific paper)
6. **Implementation priority** (ordered by impact/effort ratio)

Check `notes/project_state.md` for context from prior skill invocations. Use `search_papers` to verify what comparable recent papers include in their test batteries.
